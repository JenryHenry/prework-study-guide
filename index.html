HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user
  • Semantic elements like header, footer, and aside describe website sections for accessibility
  • Elements that have other elements nested inside them are called Parent Elements
  • Elements nested inside Parent Elements are called Child Elements

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

Javascript

  • Javascript is case sensitive.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A JS identifier usually starts with a letter, underscore, or dollar sign.
  • In a statement like 'let x = 42', the 'let x' part is called a declaration, and the '= 42' part is called an initializer.
  • The declaration allows the variable to be accessed later in code without throwing a 'ReferenceError'.
  • The initializer assigns a value to the variable.
  • Variables may hold values that have different data types:
  • STRINGS are defined using single or double quotation marks: let myVariable = 'Bob';
  • NUMBERS don't have quotes around them: let myVariable = 10;
  • BOOLEANS are True/False: let myVariable = true;
  • ARRAYS are structures that allow you to store multiple values in a single reference: let myVariable = [1, 'Bob', 'Steve', 10]
  • Refer to each member of the array like this: myVariable[1], myVariable['Bob']
  • OBJECTS can be anything. Everything in JS is an object and can be stored in a variable: let myVariable = document.querySelector('h1');
  • Comments are snippets of text that can be added along with code. The browser ignores text marked as comments. EX: /* Everything in between is a comment. */
  • If your comment contains no line breaks, you can put it behind two slashes: // This is a comment
  • An operator is a mathematical symbol that produces a result based on two values.
  • Addition: Adds two numbers together or combines two strings: 6 + 9; 'Hello' + 'World';
  • Subtraction, Multiplication, Division: -, *, / are their respective symbols. Basic Math.
  • Assignment assigns a value to a variable. let myVariable = 'Bob';
  • Strict equality performs a test to see if two values are equal and of the same data type. It returns a Boolean result: If the value of myVariable is 3, let myVariable = 3, and you want to test if myVariable is equal to 3, you would type myVariable === 3 , and it would return a 'true' result.
  • Not, Does-not-equal returns the logically opposite value of what it precedes. It turns a true into a false, etc. When used alongside the Equality operator, the negation operator tests whether two values are NOT equal. !, !== Assuming myVariable = 3, !(myVariable === 3); will return false.
  • Conditionals are used to test if an expression returns true or not. A very common form is the 'if...else' statement: let iceCream = "chocolate"; if (iceCream === "chocolate") { alert("Yay, I love chocolate ice cream!"); } else { alert("Awww, but chocolate is my favorite..."); }
  • Functions are a way of packaging functionality that you wish to reuse. It's possible to define a body of code as a function that executes when you call the function name in your code. This helps save time by not repeatedly writing the same code. EX: let myVariable = document.querySelector("h1");
